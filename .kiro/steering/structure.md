# プロジェクト構造

## ルートディレクトリ
```
├── articles/           # Markdown形式の技術記事
├── books/             # 本のコンテンツ（将来的な利用予定）
├── .kiro/             # Kiro AI アシスタント設定
├── .vscode/           # VS Code ワークスペース設定
├── package.json       # Node.js プロジェクト設定
└── package-lock.json  # 依存関係ロックファイル
```

## コンテンツの整理

### Articlesディレクトリ (`/articles/`)
- すべての技術記事をMarkdownファイルとして格納
- 各記事は独立した `.md` ファイル
- ファイル名は説明的でケバブケース形式
- gitでディレクトリを維持するため `.keep` ファイルを含める

### Booksディレクトリ (`/books/`)
- 将来の本コンテンツ用に予約
- 現在は `.keep` ファイルのみ
- 本機能実装時に構造を決定予定

## ファイル命名規則
- **記事**: 説明的なケバブケース形式のファイル名
  - 例: `synology-nas-no-space-left-btrfs-metadata.md`
- **ディレクトリ**: 小文字、説明的な名前
- **Keepファイル**: gitで空ディレクトリを保持するため `.keep` を使用

## コンテンツガイドライン
- すべてのMarkdownファイルには適切なYAML frontmatterを含める
- 視覚的な魅力のため適切な絵文字アイコンを使用
- 発見しやすさのため関連トピックでタグ付け
- 日本語と英語の両方のコンテンツをサポート
- 適切なシンタックスハイライトでコードブロックを含める
