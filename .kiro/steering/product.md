# Zennコンテンツ制作プロジェクト

## プロダクト概要
このプロジェクトは、エンジニアの知見共有とコミュニティ貢献を目的としたZennプラットフォーム連携リポジトリです。GitHub連携により、ローカル環境での執筆から自動公開まで一貫したワークフローを提供します。

### Zennプラットフォームの価値
- **知見の共有**: 開発の知見やアイデアを気軽に記事化し、markdownエディターで快適に執筆
- **収益化**: 技術書の販売やバッジによる収益機会（現金やAmazonギフトカードで還元）
- **コミュニティ貢献**: エンジニア同士の助け合い文化への参加
- **キャリア価値**: 情報発信によるキャリア向上と正当な評価の獲得
- **対価の獲得**: 地道に有益な知見を発信し、コミュニティに貢献するエンジニアが安心して対価を得られる場

## Zennが目指す世界
情報発信を通じてエンジニア自身とコミュニティ全体の価値向上を目指しています。全てのエンジニアが情報発信による価値を最大化し、キャリアを通じて大きな価値を得られることを願っています。

## コンテンツ品質基準（Zennガイドライン準拠）

### 推奨する記事の特徴
- **タイトルと内容の一致**: 読者の期待に応える正確なタイトルで信頼感を提供
- **概要の明記**: 冒頭で記事の内容と得られる知識を明示し、読者をスムーズに本題へ導く
- **コンテキストの明確化**: 想定環境や再現条件の詳細な記述で再現性の高い情報を提供
- **対象読者の設定**: 適切な説明レベルの判断基準を明確にする
- **完成度の重視**: 最後まで書き上げてから公開し、読者に良い印象を与える
- **個人の経験・考察**: 公式情報に加えた実体験や独自の視点で価値を提供

### 避けるべき内容
- **広告・宣伝目的**: 製品や採用の宣伝が主目的の投稿（知見共有が主であること）
- **誇張的なタイトル**: クリックベイト的な大げさな表現で読者をがっかりさせる内容
- **著作権侵害**: 適切な引用範囲を超えた他者の著作物使用
- **AI生成コンテンツの乱造**: 
  - 内容の正確性を確認せずに記事を投稿すること
  - 製品やサービスの宣伝を主な目的として記事を投稿すること
  - フォロワー獲得やスコア上昇を主な目的として記事を乱造すること
  - 外部サイトへ流入させることを主な目的として記事を投稿すること

## コンテンツ戦略

### 記事タイプ（type指定）
- **tech**: プログラミング、ソフトウェア、ハードウェア、インフラなどの技術に関する具体的な内容
  - 例: Cline+Claudeを使う開発でハマった点
  - 例: Swiftの○○でつまずいたときの対処法
  - 例: GitHub Actionsで自動デプロイ
  - 例: AWS IAMポリシーの設定方法
  - 例: ReactとVueのパフォーマンス比較検証

- **idea**: キャリア、マネジメントなど技術に直接関係しないトピックや、技術に関する抽象的な考え方・情報のまとめ記事
  - 例: OpenAI最新リリースのまとめ
  - 例: マネジメントで気をつけていること
  - 例: おすすめの技術書/記事
  - 例: エンジニアのキャリアパス
  - 例: ○○というサービスを作りました

### 推奨トピック領域
- インフラ・DevOps・システム運用
- Web開発技術・フレームワーク
- 開発ツール・環境構築
- 問題解決・トラブルシューティング

### 対象読者層
- 初級〜中級の開発者
- 特定技術の学習者  
- 実践的な解決策を求める技術者
- 同じ課題に直面している開発者

## 記事作成のベストプラクティス

### Front Matter設定
```yaml
---
title: "具体的で検索しやすいタイトル"
emoji: "😸"              # 内容に関連する絵文字1文字
type: "tech"             # techまたはidea
topics: ["docker", "aws", "devops"]  # 関連技術タグ（最大5個）
published: true          # 公開設定
published_at: 2050-06-12 09:03  # オプション: 公開予約日時
---
```

### 構成の推奨パターン
1. **概要・背景**: 記事の目的と想定読者
2. **前提条件**: 必要な知識や環境
3. **本文**: 段階的で再現可能な手順
4. **まとめ**: 要点の整理と次のステップ

### コード記述の注意点
- シンタックスハイライト用の言語指定必須
- ファイル名付きコードブロックの活用
- 実行可能で完全なコード例の提供
- 適切な日本語コメントの追加

## 収益化・評価の仕組み

### バッジシステム
- 読者からの有料バッジで著者を応援
- 現金やAmazonギフトカードでの還元

### 本の販売
- 技術書として章立てされたコンテンツ
- 200-5000円での価格設定（100円単位）
- 無料公開も選択可能

## コミュニティ参加の心構え

### 基本姿勢
- どんなに小さくても良いのでコミュニティに貢献する内容にする
- 常に読み手がいることを忘れず、他者への敬意を払う
- 建設的なコミュニケーションを意識する
- 公開の場であることを常に意識する

### コメント時の注意
- 記事の著者へ最大限の敬意を払う
- 相手への配慮を忘れずに建設的なコメントを心がける
- コメントもコミュニティへの貢献の一つであることを意識する
